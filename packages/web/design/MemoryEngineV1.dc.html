Hybrid retrieval ranking weights

created Jun 18, 2026·1,284 tokens· relevance 0.94

Hybrid search blends pgvector cosine similarity with full-text ts_rank_cd scores. The final ranking weights semantic relevance at 0.6 and lexical match at 0.4, normalized per query so neither signal dominates short keyword lookups.

Re-ranking runs inside a single SQL CTE — no round-trip to the application layer.

sql rank.sql 
SELECT id, content,
  0.6 * (1 - (embedding <=> $1))
  + 0.4 * ts_rank_cd(tsv, query) AS score
FROM memories, plainto_tsquery($2) query
WHERE tsv @@ query
ORDER BY score DESC
LIMIT 20;
